🌳 Git - Version Control

ระบบจัดการเวอร์ชันไฟล์ที่ทรงพลังสำหรับทีม

📖 Git คืออะไร?

Git คือระบบจัดการเวอร์ชัน (Version Control System) ที่ช่วยให้นักพัฒนา สามารถติดตามการเปลี่ยนแปลงของโค้ด จัดการโปรเจคแบบทีม และกลับไปยังเวอร์ชันเก่าได้อย่างง่ายดาย

Git ถูกสร้างโดย Linus Torvalds (ผู้สร้าง Linux) และกลายมาเป็นมาตรฐาน ของระบบ Version Control ในอุตสาหกรรมการพัฒนาซอฟต์แวร์

⚙️ Git Configuration (ตั้งค่าครั้งแรก)

หลังจากติดตั้ง Git แล้ว ให้ตั้งค่าชื่อและอีเมลของคุณ:

☝️ ตั้งค่าชื่อผู้ใช้
git config --global user.name "Your Name"
☝️ ตั้งค่าอีเมล
git config --global user.email "your.email@example.com"
☝️ ตรวจสอบการตั้งค่า
git config --list

🚀 Git Commands พื้นฐาน

1️⃣ Clone Repository

ดาวน์โหลดโปรเจคจาก GitHub

git clone https://github.com/username/repo.git

2️⃣ Check Status

ตรวจสอบสถานะของไฟล์

git status

3️⃣ Add Files

เพิ่มไฟล์ที่ต้องการ Commit

git add .
// เพิ่มไฟล์ทั้งหมด
git add filename.txt
// เพิ่มไฟล์เดียว

4️⃣ Commit Changes

บันทึกการเปลี่ยนแปลง

git commit -m "Description of changes"

5️⃣ Push to Remote

อัพโหลดโค้ดไปยัง GitHub

git push origin main

6️⃣ Pull Latest Changes

ดาวน์โหลดการเปลี่ยนแปลงล่าสุด

git pull origin main

🌿 Branching (สาขา)

Branch ช่วยให้คุณทำงานแยกจากกัน โดยไม่กระทบต่อเวอร์ชันหลัก

📌 สร้าง Branch ใหม่
git checkout -b feature/new-feature
📌 ดูรายการ Branch ทั้งหมด
git branch -a
📌 สลับไป Branch อื่น
git checkout branch-name
📌 ลบ Branch
git branch -d branch-name
📌 Merge Branch เข้ากับ main
git merge feature/new-feature

📊 Git Workflow (ขั้นตอนการทำงาน)

Working Directory → Staging Area → Repository

ระบบ Git มี 3 ส่วนหลัก:

ส่วน คำอธิบาย Command
Working Directory ไฟล์ที่คุณแก้ไข -
Staging Area ไฟล์ที่เตรียมพร้อม commit git add
Repository บันทึกโปรเจค (History) git commit

📝 Commit Best Practices

  • Commit อย่างบ่อย: ทำให้ History ชัดเจน
  • แสดง Message ที่ชัดเจน: เพื่อให้ผู้อื่นเข้าใจ
    git commit -m "Fix: resolve login bug"
  • ไม่ Commit ไฟล์ที่ไม่จำเป็น: ใช้ .gitignore
  • Review ก่อน Commit: ใช้ git diff

🔧 Advanced Commands

📌 ดู Commit History
git log --oneline
📌 ดู Diff ระหว่าง Files
git diff
📌 Undo Commit ล่าสุด
git reset --soft HEAD~1
📌 Stash Changes (Save temporarily)
git stash
📌 Apply Stashed Changes
git stash pop

🔄 Git & GitHub Workflow

  1. Fork หรือ Clone Repository
  2. สร้าง Branch ใหม่: git checkout -b feature
  3. แก้ไขไฟล์และ Commit: git add . && git commit -m "..."
  4. Push Branch: git push origin feature
  5. สร้าง Pull Request บน GitHub
  6. รอการ Review และ Merge

💡 Tips & Tricks

  • สร้าง .gitignore ไฟล์เพื่อ exclude ไฟล์ที่ไม่ต้องการ
  • ใช้ git alias เพื่อลัดสั้น command
  • ใช้ Git GUI Tools เช่น GitGUI, GitHub Desktop สำหรับผู้เริ่มต้น
  • ติดตั้ง Git Extensions ใน VS Code
